# Change Log

## 0.1.9

- Rebuilt the README preview layout to remove overlapping text and improve spacing.
- Added a dedicated local extension icon asset and updated the manifest to use it instead of a remote preview image.

## 0.1.8

- Intensified editor contrast with more neon yellow and phosphor orange across functions, builtins, numbers, regex, escapes, warnings, and structural highlights.
- Rebalanced the editor/workbench accents so warm highlights feel more present without removing the Matrix green foundation.

## 0.1.7

- Added more real-world file mappings and ecosystem coverage for config, lockfiles, build files, IBM i source files, and frontend tooling.
- Unified older language icons with the newer digital Matrix icon system and expanded coverage for TOML, INI, properties, Terraform, Perl, Lua, Dart, Astro, Svelte, Make-style files, and env files.
- Added more language-aware token tuning for JSON, XML/HTML, CSS, SQL, YAML-like mappings, and Markdown details.
- Improved package metadata for publication with homepage, bugs URL, icon, and stronger Marketplace keywords.

## 0.1.6

- Integrated additional custom swatches (`#D16C00`, `#9EED91`, `#EDB391`, `#ED91B3`, `#B3ED91`) into GUI and editor styling.
- Added warmer contrast to warnings, modified states, numeric tokens, regex, decorators, and selected widget accents.
- Expanded icon coverage again with icons for Go, Rust, PHP, Ruby, Kotlin, Swift, Vue, and React-oriented files.

## 0.1.5

- Added phosphor orange and warm amber accents to the editor and GUI while preserving the Matrix green core.
- Expanded syntax contrast with warmer highlights for numbers, regex, escapes, warnings, and selected semantic roles.
- Greatly increased icon theme coverage for IBM i languages, systems languages, JVM, scripting, data, and tooling files.
- Added dedicated icons for COBOL, RPG/RPGLE, CL/CLLE/CLP, DDS, C, C++, C#, Java, XML, YAML, SQL, shell, PowerShell, and Docker.

## 0.1.4

- Extended the Matrix concept beyond color into workbench behavior, iconography, and UI accents.
- Added more Matrix-specific workbench styling for hover states, quick input, testing, git decorations, terminal, and symbol colors.
- Refined the theme using analogous green-lime-yellow harmony plus sparse split-complementary red accents.
- Updated core folder/file icons to feel more digital, angular, and phosphor-neon.
- Increased perceived glow through stronger borders, widget shadows, active highlights, and luminous contrast layers.

## 0.1.3

- Enriched the GUI and editor palette using more green-lime tints, tones, and shades for better depth.
- Improved visual separation across panels, sidebars, inputs, widgets, quick input, settings, and welcome surfaces.
- Added stronger neon treatment through brighter active borders, focus states, matches, progress, and widget shadows.
- Refined extension view buttons and other secondary controls so they better match the Matrix aesthetic.
- Expanded syntax contrast with lighter greens, warmer yellow-green highlights, and selective red structural accents.

## 0.1.2

- Expanded the Matrix palette with more light-green variations for syntax highlighting.
- Reworked syntax contrast so keywords, types, functions, strings, numbers, parameters, and properties read more distinctly.
- Replaced most cyan-like accents with lime/yellow-green neon tones.
- Added small red accents for errors and structural punctuation such as some brackets and delimiters.
- Increased neon intensity across focus states, tabs, cursor, matches, and active editor chrome.

## 0.1.1

- Refined editor chrome colors for tabs, toolbar, sticky scroll, and title areas.
- Improved syntax highlighting separation across keywords, functions, types, strings, numbers, and parameters.
- Strengthened neon accents in focus, cursor, selections, and active UI states.
- Added `.gitignore` entries for `.vsix` packages and local extension test artifacts.

## 0.1.0

- Initial release of Tema Matrix.
- Added the `Tema Matrix` dark color theme.
- Added the `Tema Matrix Icons` minimal custom icon theme.
- Added base assets and documentation for local use and packaging.
